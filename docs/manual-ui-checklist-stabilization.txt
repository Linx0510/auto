Manual UI checklist (stabilization — gallery)

Run with server started (DATABASE_URL set) and browser DevTools console open.

Admin (/admin, logged in as admin)

1. Sidebar «Галерея»: таблица загружается без ошибок в консоли.
2. «Добавить фото»: модалка, отправка JPG/PNG/GIF, строка появляется, превью открывается по URL из image_url.
3. «Редактировать»: смена названия/описания/категории сохраняется; опционально новый файл — старое превью не должно соответствовать старому файлу на диске.
4. «Удалить»: подтверждение, строка исчезает.

Главная (/)

5. Секция фотогалереи (#home-gallery-grid): карточки с img; при пустой БД — сообщение «Пока нет фотографий».
6. Старые сиды без файлов в public/uploads могут давать битые картинки — ожидаемо до загрузки новых файлов.

Страница с main.js и #gallery-container (если есть в разметке)

7. После загрузки видны img при непустом image_url из API.
