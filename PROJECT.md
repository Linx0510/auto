# АвтоМобилСервис

Веб-приложение для автосервисного центра **«АвтоМобилСервис»**: публичный сайт с каталогом услуг, онлайн-записью на ремонт и административной панелью для сотрудников.

## Назначение

Система связывает клиентов автосервиса и администрацию в одном месте:

- **Клиентам** — узнать об услугах и ценах, посмотреть галерею и отзывы, зарегистрироваться и оставить заявку на обслуживание без звонка в офис.
- **Администраторам** — принимать и вести заявки, управлять каталогом услуг и контентом сайта, смотреть сводную статистику по работе сервиса.

Проект помогает сократить ручную работу на стойке приёма: заявки фиксируются в базе с данными об автомобиле и услуге, статусы обновляются в личном кабинете и админке.

## Технологии

| Компонент | Стек |
|-----------|------|
| Backend | Node.js, Express |
| База данных | PostgreSQL |
| Шаблоны страниц | EJS (рендер HTML из `views/`) |
| Авторизация | Сессии (`express-session`), хеширование паролей (`bcrypt`) |
| Обновления на сайте | Socket.IO (мгновенное обновление списка услуг на главной) |
| Загрузка файлов | Multer (фото в галерее) |
| Почта | Nodemailer (восстановление пароля) |

## Роли пользователей

### Гость (не авторизован)

Просматривает публичную часть сайта без входа в систему.

### Клиент (`client`)

Роль по умолчанию при регистрации. Доступ к личному кабинету `/profile`.

### Администратор (`admin`)

Полный доступ к админ-панели `/admin`. При первом запуске в БД создаётся учётная запись администратора (см. `database/database.js`, функция `seedAdmin`).

> В системе только две роли: `admin` и `client`. Назначение и снятие роли администратора выполняется из админ-панели; последнего администратора понизить до клиента нельзя.

## Основные действия по ролям

### Гость

| Действие | Описание |
|----------|----------|
| Просмотр главной страницы | Услуги, преимущества, контакты, блок руководителя |
| Каталог услуг | Список активных услуг с ценой и длительностью |
| Галерея | Фотографии работ, помещений, команды |
| Отзывы | Публичные одобренные отзывы клиентов |
| Регистрация / вход | Страница `/login` |
| Запись на услугу с главной | Через форму контактов (требуется авторизация для отправки заявки) |
| Восстановление пароля | Запрос ссылки на email, сброс по `/reset-password` |

### Клиент

| Действие | Описание |
|----------|----------|
| Регистрация | Email, пароль, ФИО, телефон → автоматический вход |
| Вход / выход | Сессия, выход через `/logout` |
| Создание заявки | Услуга, марка/модель авто (справочник или «другое»), год, госномер, описание проблемы, желаемая дата |
| Просмотр своих заявок | История с группировкой по статусам |
| Настройки профиля | Просмотр и редактирование данных в интерфейсе личного кабинета |

### Администратор

| Раздел | Действия |
|--------|----------|
| **Статистика** | Число заявок (всего, в ожидании, завершённых), клиентов, условная выручка по завершённым заявкам; последние заявки |
| **Заявки** | Просмотр всех заявок, фильтры (статус, даты, марка), сортировка, смена статуса |
| **Пользователи** | Список пользователей, назначение роли `admin` / `client` |
| **Услуги** | Создание, редактирование, деактивация услуг (название, описание, цена, длительность, категория) |
| **Галерея** | Загрузка, изменение и удаление изображений с подписью и категорией |

## Статусы заявок

| Статус | Значение |
|--------|----------|
| `pending` | Ожидает обработки (статус при создании) |
| `in_progress` | В работе |
| `completed` | Завершена (фиксируется время `completed_at`) |
| `cancelled` | Отменена |

## С чем помогает проект

**Для бизнеса автосервиса**

- Единая точка приёма онлайн-заявок вместо разрозненных звонков и записей.
- Актуальный прайс и описание услуг на сайте без правки вёрстки вручную.
- Учёт заявок и простая аналитика (нагрузка, завершённые работы, выручка по услугам).
- Управление визуальным контентом (галерея) и ролями сотрудников.

**Для клиентов**

- Прозрачный список услуг и цен.
- Запись на ремонт/обслуживание из браузера в любое время.
- Личный кабинет с историей обращений и текущим статусом заказа.
- Быстрый вход и восстановление доступа по email.

**Для разработки и сопровождения**

- Чёткое разделение: `repositories/`, `services/`, `routes/`, `middleware/`.
- Миграции SQL в `database/migrations/`.
- Справочник марок и моделей автомобилей для корректного заполнения заявок.

## Страницы приложения

| URL | Назначение |
|-----|------------|
| `/` | Главная (лендинг) |
| `/login` | Вход и регистрация |
| `/profile` | Личный кабинет клиента |
| `/admin` | Админ-панель |
| `/reset-password` | Установка нового пароля по ссылке из письма |
| `/logout` | Выход из учётной записи |

## Запуск

1. Скопировать `.env.example` в `.env` и указать `DATABASE_URL` для PostgreSQL.
2. Установить зависимости: `npm install`.
3. Запустить сервер: `npm run dev` (разработка) или `npm start` (продакшен).

При старте выполняется инициализация схемы БД, применение миграций и начальное наполнение (администратор, услуги, галерея, справочник авто).

## Структура проекта (кратко)

```
autoservice/
├── server.js          # Точка входа, маршруты страниц, Socket.IO
├── routes/api.js      # REST API
├── database/          # Подключение к PostgreSQL, init, миграции
├── repositories/      # Запросы к БД
├── services/          # Бизнес-логика (почта, валидация авто)
├── middleware/        # Авторизация, загрузка файлов
├── views/             # HTML-шаблоны страниц
└── public/            # CSS, JS, статика и uploads
```
